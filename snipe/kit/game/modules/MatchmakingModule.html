package snipe.kit.game.modules

class MatchmakingModule

extends Module<ClientKit, ServerKit>

This module provides client request functionality for basic matchmaking.

It is available only on Snipe Kit server.

Constructor

new (srv:ServerKit)

Methods

add (c:ClientKit, params:Params):Dynamic

kit/matchmaking.add

Adds this user to the specified matchmaking user list (bound to game type).

User has to be in the default state. Puts the user into "kit/match" state.

Note: A single user can only be in one matchmaking list. It will be automatically removed from other lists. If the user is added for the second time, his record will be replaced with the new one.

Input:

  • typeID - String. Game type ID.

Output:

  • errorCode - String. Operation error code.

Error codes:

  • ok - Operation successful.
  • listNotFound - Matchmaking list with this ID was not found.
  • wrongState - User is currently in a wrong state. State is returned in state response field.
  • Error codes defined in the action requirements.

remove (c:ClientKit, params:Params):Dynamic

kit/matchmaking.remove

Removes this user from the current matchmaking user list.

User has to be in the "kit/match" state. Puts the user into the default state.

Input:

Output:

  • errorCode - String. Operation error code.

Error codes:

  • ok - Operation successful.
  • wrongState - User is currently in a wrong state. State is returned in state response field.
  • Error codes defined in the action requirements.
